# 困于想像, 所以困于系统

# 关于回复

今天发了《外卖和货拉拉 -- 我们不必困在系统中》一文, 收到了这样一条回复.

我觉得...追求道德不道德的...系统不系统的...太虚。系统既然存在，它必然有它存在的必要，不点外卖的话，外卖小哥不就没有收入了吗，因噎废食。其实不如多体谅一下别人，但体谅这种事情不容易在于，大环境下，大家都在争抢比较，没有这个氛围：我都不够，凭什么要给你？但人心不足，永远不够。

看到这条回复, 我本能的就想回应: 这个回复的内容似乎都和本文关系不大. 首先, 这篇文章即没有什么道德考量, 也没怎么提到系统. 说到道德, 仅仅是拿 stella 关于外卖是不是有道德睱疵的问题作了说题的引子; 说到系统 ( 其实也只在标题里提到 "系统" 两个字 ) 也仅仅因为有好些个关于外卖和货拉拉的文章以 《困于系统中》为题, 我起这个名字来强调我们大可有另一个视角来这个问题. 其次, 在这篇文章里, 我也没有提倡不要点外卖, 而仅仅是说别拿外卖做日常的就餐方式, 这意思大概类似于, "别拿零食当饭吃".

如果就是想做此回应, 另写一篇文章当然是小题大作了. 我之所以要再写一篇文章, 是在细细想这个回复时, 发现了诸多有趣的疑问. 在试图回答这些疑问时, 我发现我似乎能作出些更有意义的回应来.

所以回应这个回复的过程, 其实就是理解这回复的过程.

# 追问回复

让我们来细细看这个回复, 它大概说了三个意思:

  1. 道德和系统太虚
  2. 不点外卖会导致外卖小哥没有收入, ( 所以主张不点外卖是因噎废食 ).
  3. 因为人心不足, 所以有 "我都不够, 凭什么要给你" 的氛围, 所以大家都在争抢, 所以体谅很难.

猛的一看, 会觉得是三个哪跟哪都挨着的普遍观点. 但稍微推敲一下, 就觉得这三个 "普遍观点" 尽是可疑之处.

# 关于 "太虚" 的疑问

道德太虚勉强还能让人理解, 为什么觉得系统太虚? 我写在文章的软件运行在 linux 系统里; 你看文章的手机要么是安卓系统, 要么是 ios 系统; 我们一出门, 就要使用公共交通系统; 不小心犯了事还得和公检法系统打交道. 就不攀扯太远, 外卖有外卖系统, 货拉拉有货拉拉的系统, 这些个系统你看见摸得着, 你与它交互, 它就给你反馈. 这么实在的东西, 怎么会有人觉得它太虚了?

# 关于 "没收入" 的疑问

不点外卖会导致外卖小哥没有收入, 这似乎是合乎经济学原理的道理: 需求产生就业, 扼杀了需求就减少了就业. 当然, 从最抽像的道理层面可以这样推导, 但我真的很怀疑因此就会有相信 "不点外卖外卖小哥就会没有收入". 假如因为某种不可抗力, 外卖这个行业突然消失了, 所有的外卖小哥会因此而被饿死么? 当然不会啊, 他们会去干别的工作. 就如俗话说的, 没有被尿憋死的人.

或许, 我们还可以反驳, 所谓 "没有收入" 是一种夸张的修辞, 这话真正想说的是: 对于哪些外卖小哥, 外卖是他们能选择的最好的工作, 如果不让他们去送餐, 他们只会活的更惨. 当然, 如果从 "市场会对资源进行最优的配置" 的经济学原理, 推导出 "因为外卖小哥在市场中自愿选择了外卖行业, 那么说明外卖行业对外卖小哥是最优的工作方案", 似乎也能说的通.
