# 关于有效进步的相关问题

# 有效进步

当我要说这个话题的时候, 一时间, 竟不知道要如何来表达.

所以估且这么说吧, "有效进步".

要真说起来, 很明显, "有效进步" 来源于强列的不安全感:

  • 可以很确切的说, 目前所有的生活建基于互联网产品经理的工作之上;
  • 而互联网行业因其本身的性质具有非常不稳定和性质: 这是一个完全建立在传递不平等性来获益的行业;
  • 而产品经理本身又是互联网往业中最不稳定的岗位: 这是一个完全为了传递不平等性而创建的岗位;
  • 这个结果是, 这个岗位与创造真实价值离的非常的的远. 不能创建真实价值的工作, 往往没有确定的标准, 表现出来就是: 这个岗位的随意性非常的大;
  • 所以在这个岗位上很难有硬性的积累.也就是说, 一个产品经理, 在一个公司所积累的经验, 很难传递到别下一公司.
  • 而非常不巧的是, 互联网公司的不稳定性质, 使的工作变动非常大: B 轮死已不是什么耸人听闻的事了.

所以我的焦虑可想而知.

这也是我当初从惠智出来的主要原因.

虽然当时对于互联网行业和产品经理岗位还没有产生现在这样悲观的看法, 但是有一点, 从当时看来就是非常明确的, 那个公司是不可能走很远的.

总有一天, 我所需要面对的整个市互联网人力资源市场. 在那一天, 如果我所抱持的还是惠智的那一套, 非常有可能会没有出路.

所以当时做出了去上海的决定. 总的说来, 这个决定的是相当正确的.至少对于当时所积累的技能和思路进行了验证.

但是接踵而至的 p2p 暴雷和错误的休息三个月的决定, 使我们抗风险能力一下子跌到底线了.

而后的几次饥不择食的求职, 使稳定性每况逾下. 而对于未来, 是否可以抗到一个相对稳定和竟况, 已经变得非常不确定了.

所以, 必须要在有限的时间内 有效的进步 就为得非常迫切. 只有进步才是对抗风险的最好手段.

设想, 如果当时一股作气, 把项管考下来, 现在还真不至于如此囧迫. 多样性的技能, 可以保证我们在复杂的市场环境保证稳定性.

# 有效进步之成为问题

有效进步之成为问题, 是因为, 事实上没能做到有效进步.

应该是有各种具体的细节组成了这个问题:

首先, 什么才是有效进步, 多看了哲学书, 多写了公众号文章, 算不算有效进步? 写了脚本, 配置了 vim 算不算有效进步？ 如果算, 它带来什么样的进步. 这样的进步有什么意义, 如果不算, 那它们是不够有效, 还是不够进步? 那么做什么才算得上是有效进步?

或者我们应当从什么是进步开始讨论. 只有知道了什么是进步, 我们才能进一步判断, 什么是有效的进步, 什么的进步不够有效.

如果我们回到为什么要有 有效进步 讨论上, 我们可以得到这样的结论. "有效进步" 是为了通过增加自身的知识或技能或经验, 而获得一定程度的稳定性, 以对抗可能出现的风险.

那么我们在讨论是否 进步 时, 我们其实是在讨论, 我们是否真正的增了稳定性.

好我们进一步去想这个问题, 什么样的事情产生稳定性. 或者更形象的说, 什么的事情使我们积累稳定性.

硬技能产生稳定性. 什么是硬技能. 大概有两个标准: 形成门槛和具有客观标准.

形成门槛是指: 撑握这个技能, 则可以多做一些事. 比如, 说话的技能, 掌握了, 可以说的好一些, 不掌握, 也不耽误说话. 这就不是硬技能. 软件操作, 专门学习了, 可以用的好一点, 不学习, 自己摸索也将就用, 这就不是什么硬技能.

具有客观性是指: 这个技能, 掌握的好不好, 是可以用通客观的方式显式了来的. 比如, 如果学习一门编程语言, 学到什么程度, 掌握多少, 这是可以量化的. 但是, 象是思维方式, 广而言之的方法论, 这种东西, 其实就没什么客观标准. 任何一个人都可以对自己的一套方法论讲出一个说法来.

复杂性产生稳定性. 其实更确切的说, 冗余产生程定性, 而复杂性产生冗余. 既然我们面对的是未来不可知的风险, 那么能够应对更多情况, 就比只能在一处专精更具有程定性.

当然, 对冲是一个好注意, 但是, 找到一个好对子可能是困难的, 而能够形成对冲, 就更难了. 比如, 如果学习机械加工一定是可以形成对冲的. 但是目前的环境却很难学习机械加工.

另外, 对子都是人在可知的情况下形成的认知方式. 但世界不是这样的. 所以如果真的冲着对子去了, 但有可能落个两头空.

在这一点上, 可以延伸出的是, 基础性产生稳定性. 因为基础性的, 或以在更多的地方被应用. 这就是一种产生冗余的情况.

外部性也可以产生稳性定. 这个来自于负墒. 只有在外部交流的情况下, 内部才能产生秩序.

读哲学书, 可以算到基础性上, 写公众号, 可以算到外部性上. 学习 vimScript 可以算到硬技能上. ( 但其实你想一下, 这个其实也不那么硬. 除非真能用 vimScript 做到一些别人做不到的事情. )

那么什么是有效. 读哲学书更有效, 还是写公众号更有效. 我们应该把更多的时间放在配置 vimrc 上, 还是把更多的时间, 用来摘录赫西俄德的神谱?

仅这样横向来比较, 我们什么都比较不出来. 因为,

  • 首先, 如果我们横向来进行比较, 我们其实是在对它们进行量化. 但它们本质上, 就是性质不同的事情, 性质不同的事情, 进行量化, 必然会进行过度简化. 在这种事情过度简化, 那何苦还做呢?
  • 其次, 如果我们以这种确知的理性行事, 就很难去应对以不可知为基础的风险了.

可能, 如果非卡在时间有限上, 我们真的无法定义出一个判断 有效 的方法.

# 怎么办

那么怎么办? 如何使我们的能够有效的进步, 以保正, 在真要面对风险的那一天里, 具有真正的抵抗能力.

想来想去, 似乎只有 做事 一个办法.

那问题就变成, 做什么事. 当然可以说, 使自己是其所是的事. 但是, 在这里说这样的放之四海皆准的废话, 其实没什么意思. 因为, 只说使自己是其所事, 我们依然不知道该做什么.

曾经有过这样的想法, 就是设计一个供应链金融系统, 然后自己实现之. 但是, 这个想法到现在还停留在想法上.

同样的想法, 还可以是设计一个通用积分系统, 然后自己实现之.

或者, 还可, 面向考试. 比如再试着考软考. 但是, 我们真的有这样的能力和心气完成这样的一个任务么? 你要知道, 我们的客观环境是回到无锡的家里, 只能偷摸的学习.

另一个比较可行的想法, 就是看书. 以看书的任务为主, 日拱一卒的扎扎实实的看些书. 但是, 可能就又会陷入到原来那种不知道该如何判断是否在 有效进步 的状况中了.
